Architecture Overview

MedSafe is designed as a secure, offline-first medication companion for Android. It ensures patients never miss crucial dosages by combining scheduled background intents with state persistence and local database operations.

Reliable Schema Modeling

The database relies on two interrelated SQLite tables via Android Room. All updates, such as logging compliance, run asynchronously on background threads using Kotlin coroutines. 

// Medicine.kt Entity
@Entity(tableName = "medicines")
data class Medicine(
    @PrimaryKey(autoGenerate = true) val id: Int = 0,
    val name: String,
    val dosage: String,
    val timeInMinutes: Int, 
    val type: String = "Pill", // Pill, Syrup, Injection
    val stock: Int = 0,
    val isActive: Boolean = true
)

// DoseLog.kt Entity (Logs medication status)
@Entity(tableName = "dose_logs")
data class DoseLog(
    @PrimaryKey(autoGenerate = true) val id: Int = 0,
    val medicineId: Int,
    val scheduledTime: Long,
    val takenTime: Long? = null,
    val status: String // PENDING, TAKEN, MISSED
)

Exact Time Triggering

Android restricts background execution to save battery. MedSafe uses exact alarm APIs (`setExactAndAllowWhileIdle`) along with permission verification to guarantee callbacks even if the device is in Doze mode. 

// AlarmScheduler.kt
fun scheduleReminder(medicine: Medicine) {
    val intent = Intent(context, ReminderReceiver::class.java).apply {
        putExtra("MEDICINE_ID", medicine.id)
        putExtra("MEDICINE_NAME", medicine.name)
    }
    val pendingIntent = PendingIntent.getBroadcast(
        context, medicine.id, intent,
        PendingIntent.FLAG_UPDATE_CURRENT or PendingIntent.FLAG_IMMUTABLE
    )
    val calendar = Calendar.getInstance().apply {
        val hour = medicine.timeInMinutes / 60
        val minute = medicine.timeInMinutes % 60
        set(Calendar.HOUR_OF_DAY, hour)
        set(Calendar.MINUTE, minute)
        set(Calendar.SECOND, 0)
        if (timeInMillis <= System.currentTimeMillis()) {
            add(Calendar.DAY_OF_YEAR, 1)
        }
    }
    alarmManager.setExactAndAllowWhileIdle(
        AlarmManager.RTC_WAKEUP, calendar.timeInMillis, pendingIntent
    )
}

Background Receivers

The app splits logic into separate Broadcasters for responsive state handling:

  • ReminderReceiver: Creates a new pending log, presents a high-importance notification banner, and triggers a 1-hour missed check alarm.
  • MissedDoseReceiver: Triggers 1 hour after the dose schedule. If the status remains pending, it is marked missed. It then evaluates if the user missed 3 consecutive doses to fire an Emergency warning.
  • ActionReceiver: Intercepts click broadcasts from the notification drawer, allowing users to record a dose as taken immediately without opening the main UI.
